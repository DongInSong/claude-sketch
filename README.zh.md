# ✳ claude-sketch

[English](README.md) · [한국어](README.ko.md) · [日本語](README.ja.md) · **中文**

> Claude Code 刚刚第六次读了 `fusion.py`。
> 顺便还去 `vendor/urllib3/` 逛了一圈。
> 你完全不知道。现在你知道了。

**claude-sketch** 用蜡笔画出 Claude Code 此刻在你的仓库里碰了什么：踩过哪些文件、
哪些从未打开、注意力究竟落在何处、删掉了什么、哪里失败了，以及这趟冒险一共花了
多少 token。

不需要账号、云端和配置。它只读取 Claude Code 早已写在你磁盘上的会话记录，
任何内容都不会离开这台机器。

```bash
npx claude-sketch
```

## 怎么看这张纸

**✳ 足迹** — 仓库里的每个文件，按文件夹一行排开。涂了蜡笔的格子是 Claude 踩过的文件，
虚线格子是它*擦肩而过*的邻居。`⋯ 16 ⋯` 是它压根没靠近的区段，折叠起来，免得单体仓库
变成一整面墙纸。上面的 `16 / 58 (28%)` 是以 `git ls-files` 为分母、老老实实数出来的覆盖率。

**◆ 注意力占比** — 一张**面积 = 工具调用次数**的树图，所以这张图本身就是结论。
当一个谁也没让它碰的配置文件吞掉了三分之一的画面，你不需要任何指标就知道跑偏了。
越亮越近期，越大越累积，`+3 −1` 是变动的行数。

**▨ 文件热度** — 最近 5 分钟、1 小时或整场会话，一律铺成 30 列。蜡笔越深，那一段越忙。
节奏会在这里显形：读、读、改、跑、读……或者更让人叹气的那种：读、读、读、读、读。

**边注** — 这张纸会自己写字。*“fusion.py 读取 6 次 — 多半已在上下文里”*、
*“探索依赖／缓存路径 9 次”*、*“发现 编辑 → 运行 的循环 — 健康的节奏”*。
也会夸你，它不只是来挑刺的。

**智能体** — `main ─task▶ 子智能体`，各自带着模型、拿到的任务说明、是否还在运行、
以及用了多久。模型名按家族分级：**✦ Fable** 泛着流动的金色，**◆ Opus** 是加粗强调色，
**◇ Sonnet** 是墨色，*Haiku* 轻巧利落。哪一档在烧，一眼就能看出来。
点一下，所有视图就只剩这一个智能体。

## 那个让所有人吃惊的数字

```
新输入 485 · 缓存 8.4M (100% 复用) · 输出 10.6k · 按 API 价约 $0.03
```

Claude Code 每一轮几乎都会从缓存里重放整个上下文。生的“输入 840 万 token”看着像灾难，
实则毫无意义；真正花钱的是那个 **485**。claude-sketch 把新输入和缓存流量分开，
给出复用率，并按你可以自行编辑的价目表（`~/.claude-sketch.pricing.json`）折算成
API 价格。如果你用的是订阅套餐，本来就不按 token 计费 —— 请把这个数字读作
*工作量的大小*，而不是账单。

另一个诚实的数字是**失败的调用**。返回错误的调用花掉了 token 却什么也没换回来。
它们按文件和智能体分别计数，所以那个一直报错的文件绝不会被漏掉。

## 安装

```bash
npx claude-sketch                       # 观察当前目录
npx claude-sketch --project ~/dev/app   # 观察别的项目
npx claude-sketch --port 5000 --no-open
```

想常备一份：

```bash
npm i -g claude-sketch
cd ~/dev/app && claude-sketch
```

Node 18+。**零运行时依赖** —— 全部只有五个小模块和一个 HTML 文件。

## 工作原理

Claude Code 会把会话的每个事件追加到
`~/.claude/projects/<目录 slug>/<会话ID>.jsonl`，
子智能体则写入 `<会话ID>/subagents/**/agent-<id>.jsonl`。claude-sketch：

1. **tail 这些文件** —— 不用监听库，也不用原生模块；
2. **解析 `tool_use` 条目**（Read / Grep / Glob / Edit / Write / Bash / Task…），
   并用 `tool_use.id` 去掉流式重写造成的重复；
3. **读取 `message.model` 与 `message.usage`**，因此模型与 token 是*实测*而非估算；
4. **也读命令本身** —— 文件真正被删除的地方是 Bash 里的 `rm`，
   所以被删掉的文件会带着删除线和 ✕ 出现；
5. **通过 SSE 推送到浏览器**，由几百行原生 JS 涂上蜡笔。

没有数据库，没有构建步骤，没有框架。覆盖率的分母来自 `git ls-files`。

## 它不会碍事

用真实记录测过，其中最大的会话文件有 **260MB**：

| | |
|---|---|
| 空闲 CPU（浏览器已连接） | 单核的 **0.08%** |
| 内存（一般会话） | 约 55MB |
| 内存（最差：351 个子智能体、1 万条事件） | 约 105MB |
| 首次解析那份 260MB 记录 | **1.7 秒** |
| 稳定期轮询 | 每次 0.02ms |
| 浏览器整页重绘 | 约 26ms，堆 3MB |

不可能包含工具调用、usage 或提示词的行，连 `JSON.parse` 都不会做。文件按每次 32MB
的预算读取，落后时不等下一次轮询、立刻续读。安静一阵后轮询从 600ms 退到 2.5 秒，
一有新字节就立即回到高频。

**对 Claude Code 的影响：结构上为零。** 这个工具只读 —— `lib/` 里没有任何
`writeFile`、`unlink` 或 `mkdir`。记录以 `'r'` 打开、按偏移读取，不会阻塞 Claude Code
的追加写入。服务器只绑定 `127.0.0.1`。点击文件会先询问再打开，
`POST /api/open` 会拒绝项目、主目录和临时目录之外的路径。

## 值得知道的细节

- **`n 次调用 · 0 个文件` 是真实状态，不是 bug。** Shell 命令和 task 启动并不指向文件，
  所以只跑了命令的会话在地图上没有内容。页面会直说，而不是让你盯着空格子发呆。
- **task 耗时**取两个残缺时钟中较长的那个：父记录里的 启动→结果，以及子智能体自身的
  首次→末次调用。有些记录把这两个时刻写成相差 0.1 秒，照抄的话八分钟的工作会显示成 “1s”。
- **单文件的 token 数是按工具结果大小估算的**；按智能体统计的输入／输出总量则是
  API usage 的实测值。
- **文件夹标签会去掉所有路径共有的前缀**，只在地图上方写一次，
  这样单体仓库不会每行都重复 `packages/app/...`。
- Grep/Glob 归到它们的 `path` 参数上；WebFetch/WebSearch 归入 `(web)`。

## 配置

**价格。** 在 `~/.claude-sketch.pricing.json` 放一个文件：

```json
{
  "cacheReadFactor": 0.1,
  "cacheWriteFactor": 1.25,
  "models": {
    "opus":   { "in": 15,  "out": 75 },
    "sonnet": { "in": 3,   "out": 15 },
    "haiku":  { "in": 0.8, "out": 4 },
    "fable":  null
  }
}
```

单位是每百万 token。设为 `null` 的家族会显示*未设置单价*，而不是随便猜一个 ——
包括这里查不到公开价格的 Fable。

**语言。** English / 한국어 / 日本語 / 中文，在页眉那一行里切换。
新增一种语言，就是往 `public/index.html` 的 `I18N` 里加一个对象。

**字体。** 手写体是 **Excalifont** —— Excalidraw 用的那一款 —— 已本地内置，
拉丁文字无需联网。它随 MIT 许可的 `@excalidraw/excalidraw` 包分发，但字体自身的
许可文件并未公开，转发前请自行确认。韩文字形来自 Google Fonts 的 *Gaegu*，
这是页面唯一的外部请求；删掉那行 `<link>` 即可完全离线。
想换一种笔迹？改 `--hand` 这一个变量就行。

## 为什么长这样

因为做成仪表盘就会像一份状态报告，而这并不是报告。它更像你在编辑器旁摊开的一张草稿纸
—— 别人在打字的时候，你偶尔瞥一眼，而不是坐下来研读。蜡笔的毛边、歪掉的方框、
对不齐的手写字。树图格子里放得下数字就写上，放不下就安静待着。

还有一点：看着编程智能体干活本身就挺有意思，而这件事完全不需要柱状图。
