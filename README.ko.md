# ✳ claude-sketch

[English](README.md) · **한국어** · [日本語](README.ja.md) · [中文](README.zh.md)

> 방금 Claude Code가 `fusion.py`를 여섯 번째로 읽었습니다.
> `vendor/urllib3/`도 산책하듯 둘러봤고요.
> 당신은 몰랐습니다. 이제는 압니다.

**claude-sketch**는 Claude Code가 지금 당신의 저장소에서 무엇을 만지고 있는지를
크레용으로 그립니다. 어떤 파일을 밟았고, 어떤 파일은 한 번도 열지 않았고,
주목이 실제로 어디에 쏠렸고, 무엇을 지웠고, 무엇이 실패했고,
그 모험에 토큰이 얼마나 들었는지까지.

계정도, 클라우드도, 설정도 없습니다. Claude Code가 이미 당신의 디스크에 쓰고 있는
세션 기록을 읽을 뿐이고, 아무것도 이 머신을 떠나지 않습니다.

```bash
npx claude-sketch
```

## 화면 읽는 법

**✳ 발자국** — 저장소의 모든 파일을 폴더별 한 줄로. 크레용으로 칠해진 칸은 Claude가
밟은 파일, 점선 칸은 그 옆에서 *지나쳐 간* 이웃입니다. `⋯ 16 ⋯` 은 근처에도 가지 않은
구간을 접어둔 것이라, 모노레포가 벽지처럼 변하지 않습니다. 위의 `16 / 58 (28%)` 은
`git ls-files` 기준으로 정직하게 센 커버리지입니다.

**◆ 주목 점유율** — **면적 = 도구 호출 수**인 트리맵이라, 그림 자체가 판정입니다.
아무도 건드리라고 하지 않은 설정 파일 하나가 화면의 3분의 1을 삼키고 있다면,
그게 잘못됐다는 걸 알려주는 데 지표는 필요 없습니다.
밝을수록 최근, 클수록 누적, `+3 −1` 은 움직인 줄 수입니다.

**▨ 파일 히트** — 최근 5분·1시간·세션 전체를 30칸으로. 진한 크레용일수록 바빴던
구간입니다. 여기서 리듬이 드러납니다. 읽고, 읽고, 고치고, 돌리고, 읽고… 아니면
조금 슬픈 쪽: 읽고, 읽고, 읽고, 읽고, 읽고.

**여백 메모** — 시트가 스스로 적습니다. *"fusion.py 6회 재읽기 — 이미 컨텍스트에 있을
가능성"*, *"의존성·캐시 경로 9회 탐색"*, *"수정 → 실행 루프 확인 — 건강한 리듬"*.
칭찬도 합니다. 잔소리만 하려고 있는 건 아니니까요.

**에이전트** — `main ─task▶ 서브에이전트`. 각자의 모델, 자기가 받은 지시문,
아직 도는 중인지, 얼마나 걸렸는지가 함께 나옵니다. 모델명은 계열별 위계를 갖습니다.
**✦ Fable**은 금빛으로 일렁이고, **◆ Opus**는 굵은 강조색, **◇ Sonnet**은 잉크색,
*Haiku*는 가볍고 빠르게. 어느 급이 돌고 있는지 한눈에 보입니다.
클릭하면 모든 뷰가 그 에이전트만으로 좁혀집니다.

## 다들 놀라는 숫자

```
새 입력 485 · 캐시 8.4M (100% 재사용) · 출력 10.6k · API 요금 환산 약 $0.03
```

Claude Code는 매 턴 컨텍스트의 거의 전부를 캐시에서 재생합니다. 날것의
"입력 840만 토큰"은 재앙처럼 읽히지만 아무 의미가 없고, 실제로 비용이 되는 숫자는
**485**입니다. claude-sketch는 새 입력과 캐시 트래픽을 나누고, 재사용률을 보여주고,
직접 고칠 수 있는 단가표(`~/.claude-sketch.pricing.json`)로 API 요금에 환산합니다.
구독 요금제라면 애초에 토큰당 과금이 아니니, 이 수치는 청구액이 아니라
*작업의 크기*로 읽으세요.

또 하나의 정직한 숫자는 **실패한 호출**입니다. 에러로 돌아온 호출은 토큰만 쓰고
아무것도 사지 못했습니다. 파일별·에이전트별로 세기 때문에, 계속 에러가 나는 그
파일을 놓칠 수가 없습니다.

## 설치

```bash
npx claude-sketch                       # 현재 폴더 관찰
npx claude-sketch --project ~/dev/app   # 다른 프로젝트 관찰
npx claude-sketch --port 5000 --no-open
```

계속 쓸 거라면:

```bash
npm i -g claude-sketch
cd ~/dev/app && claude-sketch
```

| 옵션 | |
|---|---|
| `-p, --project <dir>` | 관찰할 프로젝트 (기본: 현재 폴더) |
| `--port <n>` | 서비스할 포트 (기본: 4517 — **이미 쓰고 있으면 다음 빈 포트로 옮겨갑니다**) |
| `--strict-port` | 다음 포트로 옮기지 말고 그냥 실패 |
| `--host <addr>` | 바인딩 주소 (기본: `127.0.0.1`. 다른 값을 주면 경고합니다 — 기록이 네트워크에 열리니까요) |
| `--open` / `--no-open` | 브라우저 자동 열기 여부 (기본: 열기) |
| `-v, --version` · `-h, --help` | 흔한 그것들 |

Node 18+. **런타임 의존성 0** — 작은 모듈 다섯 개와 HTML 한 장이 전부입니다.

## 동작 원리

Claude Code는 세션의 모든 이벤트를
`~/.claude/projects/<폴더 슬러그>/<세션ID>.jsonl` 에,
서브에이전트는 `<세션ID>/subagents/**/agent-<id>.jsonl` 에 append합니다. claude-sketch는:

1. **그 파일들을 tail 합니다** — 워처 라이브러리도, 네이티브 모듈도 없이;
2. **`tool_use` 항목을 파싱합니다**(Read / Grep / Glob / Edit / Write / Bash / Task…).
   스트리밍 중 재기록되는 줄은 `tool_use.id` 로 중복 제거합니다;
3. **`message.model` 과 `message.usage` 를 읽습니다.** 모델과 토큰은 추정이 아니라
   *실측*입니다;
4. **명령어도 읽습니다** — 파일이 실제로 지워지는 경로는 Bash 안의 `rm` 이라서,
   삭제된 파일은 취소선과 ✕ 로 표시됩니다;
5. **SSE로 브라우저에 흘려보냅니다.** 수백 줄짜리 순수 JS가 크레용을 칠합니다.

DB 없음, 빌드 없음, 프레임워크 없음. 커버리지의 분모는 `git ls-files` 가 제공합니다.

## 방해가 되지 않습니다

실제 기록으로 측정했습니다. 가장 큰 세션 파일은 **260MB** 였습니다:

| | |
|---|---|
| 유휴 CPU (브라우저 연결 상태) | 코어의 **0.08%** |
| 메모리 (일반 세션) | 약 55MB |
| 메모리 (최악: 서브에이전트 351개, 이벤트 1만) | 약 105MB |
| 260MB 기록 최초 파싱 | **1.7초** |
| 정상 상태 폴링 | 틱당 0.02ms |
| 브라우저 전체 리렌더 | 약 26ms, 힙 3MB |

도구 호출·usage·프롬프트를 담을 수 없는 줄은 아예 `JSON.parse` 하지 않습니다.
파일은 틱당 32MB 예산으로 읽되, 밀려 있으면 다음 폴링을 기다리지 않고 즉시 이어갑니다.
조용해지면 폴링이 600ms → 2.5초로 물러났다가 첫 바이트에 바로 복귀합니다.

**Claude Code에 미치는 영향: 구조적으로 없음.** 이 도구는 오직 읽기만 합니다 —
`lib/` 어디에도 `writeFile`·`unlink`·`mkdir` 이 없습니다. 기록은 `'r'` 로 열어
오프셋으로 읽으므로 Claude Code의 append를 막지 않습니다. 서버는 기본적으로 `127.0.0.1` 에만
바인딩합니다. 파일을 클릭하면 열기 전에 먼저 물어보고, `POST /api/open` 은
프로젝트·홈·임시 폴더 밖의 경로를 거부합니다.

## 알아두면 좋은 것들

- **`호출 n회 · 파일 0개` 는 버그가 아니라 실제 상태입니다.** 셸 명령과 task 실행은
  파일을 가리키지 않아서, 명령만 돌린 세션은 지도에 표시할 게 없습니다. 빈 격자를
  멍하니 보게 두는 대신 그 사실을 적어줍니다.
- **task 소요 시간**은 두 개의 불완전한 시계 중 긴 쪽입니다: 부모 기록의 생성→결과,
  그리고 서브에이전트 자신의 첫 호출→마지막 호출. 어떤 기록은 이 두 시각을 0.1초
  차이로 적어두는데, 그대로 쓰면 8분 작업이 "1s"로 나옵니다.
- **파일별 토큰은 도구 결과 크기 기반 추정치**입니다. 에이전트별 입출력 합계는
  API usage 실측값입니다.
- **폴더 라벨은 모든 경로가 공유하는 접두어를 떼어냅니다.** 지도 위에 한 번만 적으므로
  모노레포에서 `packages/app/...` 이 줄마다 반복되지 않습니다.
- Grep/Glob은 `path` 인자 기준으로 귀속되고, WebFetch/WebSearch는 `(web)` 으로 모입니다.

## 설정

**단가.** `~/.claude-sketch.pricing.json` 에 파일을 두세요:

```json
{
  "cacheReadFactor": 0.1,
  "cacheWriteFactor": 1.25,
  "models": {
    "opus":   { "in": 15,  "out": 75 },
    "sonnet": { "in": 3,   "out": 15 },
    "haiku":  { "in": 0.8, "out": 4 },
    "fable":  null
  }
}
```

백만 토큰당 단가입니다. `null` 인 계열은 추측하지 않고 *단가 미설정* 으로 표시됩니다.
공시가를 확인할 수 없는 Fable이 여기 해당합니다.

**언어.** English / 한국어 / 日本語 / 中文, 헤더의 언어 줄에서 고릅니다.
추가하려면 `public/index.html` 의 `I18N` 에 객체 하나만 넣으면 됩니다.

**서체.** 손글씨는 **Excalifont** — Excalidraw가 쓰는 그 서체 — 를 로컬 번들로
넣어 라틴 문자에는 네트워크가 필요 없습니다. MIT 라이선스인
`@excalidraw/excalidraw` 패키지 안에 들어 있지만 폰트 자체의 라이선스 파일은
공개돼 있지 않으니, 재배포 전에 확인하세요. 한글은 Google Fonts의 *Gaegu* 를 쓰며
이것이 페이지의 유일한 외부 요청입니다. `<link>` 한 줄을 지우면 완전 오프라인이 됩니다.
다른 손글씨를 원하면 `--hand` 변수 한 줄만 바꾸세요.

## 왜 이런 모습인가

대시보드로 만들었다면 상태 보고서처럼 보였을 텐데, 이건 보고서가 아니기 때문입니다.
다른 누군가가 타이핑하는 동안 에디터 옆에 펼쳐두는 연습장에 가깝습니다 —
정독하는 게 아니라 힐끗 보는 것. 크레용 자국, 삐뚤빼뚤한 상자, 줄이 딱 맞지 않는 손글씨.
트리맵 칸에 숫자가 들어갈 자리가 있으면 적고, 없으면 조용히 비웁니다.

그리고 하나 더 — 코딩 에이전트가 일하는 걸 지켜보는 건 꽤 재미있는데,
그 사실에 막대 그래프는 전혀 필요하지 않았습니다.
