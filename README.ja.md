## 👀 見えるもの

- 👣 **足あと** — リポジトリの全ファイル。塗り = 踏んだ、破線 = 通り過ぎた、`⋯16⋯` = 近づきもせず。カバレッジはそのセッションが実際に作業したリポジトリ基準 — 追跡ファイルに、無視されていないファイルも合わせて。
- 🎯 **注目シェア** — **面積 = ツール呼び出し数**のツリーマップ。1 ファイルが画面の 1/3 を食べていたら、その絵が警告。
- 🔥 **ファイル熱量** — 5分 / 1時間 / セッション全体。読む-読む-直す-走らせる…あるいは読む-読む-読む-読む。😬
- 📂 **フォルダ切り替え** — Claude Code が作業した全フォルダをヘッダーから選択（パスは記録から復元）。再起動不要。
- 🧠 **エージェント** — `main ─task▶ サブエージェント`。モデル・指示文・状態・所要時間つき。クリックでそのエージェントだけに絞り込み。
- 📝 **余白メモ** — 「6 回再読込、すでに文脈にあるはず」「`vendor/` を 9 回探索」「編集 → 実行のループ 👍」。
- ✂️ **削除と差分** — Bash の `rm` は取り消し線で、編集は `+12 −3` で。

## 💸 誰も予想しない数字

```
新規入力 485 · キャッシュ 8.4M (100% 再利用) · 出力 10.6k · API 料金換算 約$0.03
```

Claude は毎ターン文脈のほぼ全部をキャッシュから再生します。「入力 840 万トークン」😱 は
無意味で、実際に費用になるのは **485**。単価は `~/.claude-sketch.pricing.json` で編集。
サブスクならトークン課金ではないので、請求額ではなく*仕事の大きさ*として読んでください。
失敗した呼び出しも数えます — 純粋な無駄なので。❌

## ⚙️ 仕組み

Claude Code はすでに全イベントを `~/.claude/projects/<スラッグ>/<セッション>.jsonl` に
（サブエージェントは `<セッション>/subagents/**` に）書いています。`CLAUDE_CONFIG_DIR` を
設定していればそちらです。claude-sketch はそれを監視しつつ
tail し、`tool_use` + `message.model` + `message.usage` を取り出して SSE で流すだけ。

**推定ではなく実測。** DB なし・ビルドなし・フレームワークなし、**依存ゼロ**。📦

## 🔒 プライバシー

大事なところはリードオンリー — プロジェクトや `~/.claude` の下には何も書きません。
書くのは一時ディレクトリのマーカー1つだけで、再起動時にタブが開いているか知るためのものです。
`127.0.0.1` にバインドし、ドメイン名で届いたリクエストや別オリジンからのリクエストは拒否します。
ファイルをクリックすると**まず確認**します。外部リクエストはゼロ — フォントも同梱なので機内でも動きます。🏠

## 🎨 ディテール

- **言語**: 🇬🇧 🇰🇷 🇯🇵 🇨🇳 — ヘッダーの 1 行、追加は `I18N` にオブジェクト 1 つ。
- **書体**: Excalifont（ラテン）・Gaegu（ハングル）・Klee One（日本語）・Ma Shan Zheng（中国語） —
  いずれも **SIL Open Font License 1.1**、サブセットして同梱。カバー範囲と再生成の手順は
  [`public/fonts/README.md`](public/fonts/README.md)。`--hand` CSS 変数で差し替え可。
- **`呼び出し n 回 · ファイル 0`** はバグではありません — シェルコマンドはファイルを指さないので。
- task の所要時間 = *起動→結果* と *エージェント自身の 最初→最後* の**長い方**
  （記録によってはこの 2 点が 0.1 秒差 🙃）。
