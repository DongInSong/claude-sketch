# ✳ claude-sketch

[English](README.md) · [한국어](README.ko.md) · **日本語** · [中文](README.zh.md)

> たった今、Claude Code が `fusion.py` を 6 回目に読みました。
> ついでに `vendor/urllib3/` まで散歩してきたようです。
> あなたは気づいていませんでした。もう気づけます。

**claude-sketch** は、Claude Code が今このリポジトリで何に触れているかをクレヨンで
描き出します。どのファイルを踏んだか、どれを一度も開いていないか、注目が実際に
どこへ向かったか、何を消したか、何が失敗したか、そしてその冒険にトークンが
どれだけかかったか。

アカウントもクラウドも設定も不要です。Claude Code がすでにディスクへ書いている
セッション記録を読むだけで、何ひとつこのマシンから出ていきません。

```bash
npx claude-sketch
```

## 画面の読み方

**✳ 足あと** — リポジトリの全ファイルを、フォルダごとに 1 行で。クレヨンで塗られた枠は
Claude が踏んだファイル、破線の枠はその隣で*通り過ぎられた*ファイルです。`⋯ 16 ⋯` は
近づきもしなかった区間を畳んだもので、モノレポが壁紙になるのを防ぎます。上の
`16 / 58 (28%)` は `git ls-files` を分母にした正直なカバレッジです。

**◆ 注目シェア** — **面積 = ツール呼び出し数**のツリーマップなので、絵そのものが判定です。
誰も触れと言っていない設定ファイルが画面の 3 分の 1 を飲み込んでいるなら、
何かがおかしいと知るのに指標は要りません。
明るいほど最近、大きいほど累積、`+3 −1` は動いた行数です。

**▨ ファイル熱量** — 直近 5 分・1 時間・セッション全体を 30 列で。濃いクレヨンほど
忙しかった区間です。ここにリズムが現れます。読む、読む、直す、走らせる、読む……
あるいは少し悲しい方: 読む、読む、読む、読む、読む。

**余白メモ** — シートが自分で書き込みます。*「fusion.py を 6 回再読込 — すでに文脈に
あるはず」*、*「依存 / キャッシュのパスを 9 回探索」*、*「編集 → 実行のループを確認 —
健全なリズム」*。褒めることもあります。小言だけのための道具ではありません。

**エージェント** — `main ─task▶ サブエージェント`。それぞれのモデル、受け取った指示文、
まだ動いているか、どれだけかかったかが並びます。モデル名には系列の序列があります。
**✦ Fable** は金色に揺らぎ、**◆ Opus** は太いアクセント、**◇ Sonnet** はインク色、
*Haiku* は軽やかに。どの階層が動いているか一目で分かります。
クリックすれば、すべてのビューがそのエージェントだけに絞られます。

## 誰もが驚く数字

```
新規入力 485 · キャッシュ 8.4M (100% 再利用) · 出力 10.6k · API 料金換算 約$0.03
```

Claude Code は毎ターン、文脈のほとんどをキャッシュから再生します。生の
「入力 840 万トークン」は災害のように読めますが意味はなく、実際に費用になるのは
**485** の方です。claude-sketch は新規入力とキャッシュ通信を分け、再利用率を示し、
自分で編集できる単価表（`~/.claude-sketch.pricing.json`）で API 料金に換算します。
サブスクリプションならそもそもトークン課金ではないので、この数字は請求額ではなく
*仕事の大きさ*として読んでください。

もうひとつの正直な数字が**失敗した呼び出し**です。エラーで返ってきた呼び出しは
トークンだけを使って何も買えていません。ファイル別・エージェント別に数えるので、
エラーを繰り返しているファイルを見逃せません。

## インストール

```bash
npx claude-sketch                       # 現在のフォルダを監視
npx claude-sketch --project ~/dev/app   # 別のプロジェクトを監視
npx claude-sketch --port 5000 --no-open
```

手元に置いておくなら:

```bash
npm i -g claude-sketch
cd ~/dev/app && claude-sketch
```

| オプション | |
|---|---|
| `-p, --project <dir>` | 監視するプロジェクト（既定: カレントディレクトリ） |
| `--port <n>` | 待ち受けポート（既定: 4517 — **使用中なら次の空きポートへ移ります**） |
| `--strict-port` | 次のポートへ移らず、そのまま失敗させる |
| `--host <addr>` | バインドするアドレス（既定: `127.0.0.1`。それ以外は警告します。記録がネットワークに開くためです） |
| `--open` / `--no-open` | ブラウザを開く / 開かない（既定: 開く） |
| `-v, --version` · `-h, --help` | いつものやつ |

Node 18+。**ランタイム依存ゼロ** — 小さなモジュール 5 つと HTML 1 枚だけです。

## 仕組み

Claude Code はセッションのすべてのイベントを
`~/.claude/projects/<フォルダのスラッグ>/<セッションID>.jsonl` に、
サブエージェントを `<セッションID>/subagents/**/agent-<id>.jsonl` に追記します。
claude-sketch は:

1. **それらのファイルを tail します** — ウォッチャーライブラリもネイティブモジュールもなし;
2. **`tool_use` を解析します**（Read / Grep / Glob / Edit / Write / Bash / Task…）。
   ストリーミング中に書き直される行は `tool_use.id` で重複排除します;
3. **`message.model` と `message.usage` を読みます。** モデルとトークンは推定ではなく
   *実測*です;
4. **コマンドも読みます** — ファイルが実際に消えるのは Bash の中の `rm` なので、
   削除されたファイルは取り消し線と ✕ で表示されます;
5. **SSE でブラウザへ流します。** 数百行のバニラ JS がクレヨンを塗ります。

DB なし、ビルドなし、フレームワークなし。カバレッジの分母は `git ls-files` が担います。

## 邪魔をしません

実際の記録で計測しました。最大のセッションファイルは **260MB** です:

| | |
|---|---|
| アイドル CPU（ブラウザ接続時） | 1 コアの **0.08%** |
| メモリ（通常のセッション） | 約 55MB |
| メモリ（最悪: サブエージェント 351、イベント 1 万） | 約 105MB |
| 260MB 記録の初回解析 | **1.7 秒** |
| 定常時のポーリング | 1 ティック 0.02ms |
| ブラウザの全再描画 | 約 26ms、ヒープ 3MB |

ツール呼び出し・usage・プロンプトを含み得ない行は `JSON.parse` すらしません。
ファイルは 1 ティック 32MB の予算で読み、遅れている間は次のポーリングを待たずに
すぐ続けます。静かになるとポーリングは 600ms → 2.5 秒に下がり、最初の 1 バイトで戻ります。

**Claude Code への影響: 構造上ゼロ。** このツールは読むだけで、`lib/` のどこにも
`writeFile`・`unlink`・`mkdir` はありません。記録は `'r'` で開いてオフセット読みするため、
Claude Code の追記を妨げません。サーバーは既定で `127.0.0.1` にのみバインドします。
ファイルをクリックすると開く前に必ず確認し、`POST /api/open` はプロジェクト・ホーム・
一時ディレクトリの外のパスを拒否します。

## 知っておくとよいこと

- **`呼び出し n 回 · ファイル 0` はバグではなく実際の状態です。** シェルコマンドと task
  起動はファイルを指さないため、コマンドだけを走らせたセッションは地図に何も載りません。
  空のグリッドを眺めさせる代わりに、その旨を書きます。
- **task の所要時間**は 2 つの不完全な時計のうち長い方です: 親記録の 起動→結果 と、
  サブエージェント自身の 最初→最後 の呼び出し。記録によってはこの 2 点が 0.1 秒差で
  書かれており、そのままでは 8 分の作業が「1s」になってしまいます。
- **ファイル別トークンはツール結果のサイズからの推定値**です。エージェント別の入出力
  合計は API usage の実測値です。
- **フォルダ名は全パスに共通する接頭辞を落とします。** 地図の上に一度だけ出るので、
  モノレポで `packages/app/...` が毎行繰り返されません。
- Grep/Glob は `path` 引数に、WebFetch/WebSearch は `(web)` にまとめられます。

## 設定

**単価。** `~/.claude-sketch.pricing.json` を置いてください:

```json
{
  "cacheReadFactor": 0.1,
  "cacheWriteFactor": 1.25,
  "models": {
    "opus":   { "in": 15,  "out": 75 },
    "sonnet": { "in": 3,   "out": 15 },
    "haiku":  { "in": 0.8, "out": 4 },
    "fable":  null
  }
}
```

100 万トークンあたりの単価です。`null` の系列は推測せず*単価未設定*と表示します。
公開価格を確認できない Fable がこれに当たります。

**言語。** English / 한국어 / 日本語 / 中文。ヘッダーの言語列から選びます。
追加は `public/index.html` の `I18N` にオブジェクトを 1 つ足すだけです。

**書体。** 手書きは **Excalifont** — Excalidraw が使うあの書体 — をローカルに同梱し、
ラテン文字にネットワークは不要です。MIT ライセンスの `@excalidraw/excalidraw`
パッケージに同梱されていますが、フォント自体のライセンスファイルは公開されていないため、
再配布前に確認してください。ハングルは Google Fonts の *Gaegu* を使い、これがページ唯一の
外部リクエストです。`<link>` を 1 行消せば完全オフラインになります。
別の手書きにしたければ `--hand` 変数を 1 行変えるだけです。

## なぜこの見た目なのか

ダッシュボードにすればステータスレポートに見えたはずで、これはレポートではないからです。
別の誰かがタイプしている間、エディタの横に開いておく計算用紙に近いもの — じっくり読む
ものではなく、ちらりと見るもの。クレヨンの擦れ、歪んだ四角、きっちり揃わない手書き。
ツリーマップのマスに数字が入る余地があれば書き、なければ黙っています。

もうひとつ — コーディングエージェントが働くのを眺めるのは純粋に面白く、その事実に
棒グラフはまったく必要ありませんでした。
